You are an expert in Answer Set Programming (ASP).

Given a curated mate-in-one base fragment (board state plus helper predicates that derive legal moves), write a complete Clingo program that determines whether the side to move has a mate in one. Copy the base fragment verbatim at the top of your output, then extend it with the reasoning layer.

A mate in one occurs if a legal move by the side to move leaves the opponent king in check and the opponent has no legal reply that removes the check.

Below are representative ASP examples that illustrate the structures you should imitate.

Example 1: Graph coloring

Input facts:
node(a). node(b). node(c).
edge(a,b). edge(b,c).

ASP program:

color(red;green;blue).

1 { assign(N,C) : color(C) } 1 :- node(N).

:- assign(X,C), assign(Y,C), edge(X,Y).

#show assign/2.

---

Example 2: Reachability in a graph

Input facts:
edge(a,b).
edge(b,c).
start(a).

ASP program:

reachable(X) :- start(X).
reachable(Y) :- reachable(X), edge(X,Y).

#show reachable/1.

Example 3: ASP encoding for detecting a constraint violation:

Input facts:
student(alice).
student(bob).

course(cs).
course(math).

enrolled(alice,cs).
enrolled(alice,math).

ASP program:

:- enrolled(S,C1), enrolled(S,C2), C1 != C2, conflict(C1,C2).

#show enrolled/2.

Example ASP encoding for generating choices:

move(left;right;up;down).

1 { action(A) : move(A) } 1.

#show action/1.

Input predicates already exist inside the base fragment — do NOT redefine them:
to_move(Color).
placement(Color,Piece,Id,Row,Col).
legal_move(Id,R1,C1,R2,C2).

Board coordinates range from 1..8.

Your program must:
	•	generate legal moves and capture the resulting positions,
	•	represent attacks between pieces,
	•	detect check on the opponent king, and
	•	ensure the opponent has no legal move that escapes check (king moves, interpositions, captures).

Output the mating move using an appropriate predicate (e.g., mate_move/5).

Only output the ASP program.
